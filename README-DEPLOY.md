# ARKAN STORE — دليل النشر والربط (Railway + Telegram + TronGrid)

## ما الذي تنشره
```
deploy/
├── server.js          ← السيرفر: المتجر + الطلبات + مراقبة المحفظة + البوت
├── package.json
└── public/
    └── index.html     ← المتجر (واجهة LuxAlgo + شريط الأسعار الحي + الباقات)
```

---

## 1) تجهيز بوت Telegram (5 دقائق)

1. افتح **@BotFather** → `/newbot` → اختر الاسم واليوزر (مثلًا `ArkanAI_Access_Bot`).
2. انسخ الـ **TOKEN**.
3. افتح **@userinfobot** وأرسل أي رسالة → انسخ **ID** الخاص بك (هذا `TELEGRAM_ADMIN_ID`).
4. أنشئ قناة الأعضاء الخاصة (Private Channel) وأضف البوت فيها **Admin** بصلاحية *Invite Users via Link*.
5. لمعرفة معرّف القناة: حوّل أي رسالة من القناة إلى **@userinfobot** → يظهر رقم مثل `-1001234567890` (هذا `MEMBERS_CHANNEL_ID`).

## 2) النشر على Railway

1. ادفع مجلد `deploy/` إلى GitHub repo (أو استبدل ملفات مشروعك الحالي `arkan-ai-site-production` بها — نفس الخدمة، نفس الرابط).
2. Railway → **Variables** أضف:

| المتغير | القيمة |
|---|---|
| `WALLET_ADDRESS` | عنوان محفظتك USDT TRC20 (يبدأ بـ T) |
| `TELEGRAM_BOT_TOKEN` | توكن BotFather |
| `TELEGRAM_BOT_USERNAME` | يوزر البوت بدون @ |
| `TELEGRAM_ADMIN_ID` | معرّفك الرقمي |
| `MEMBERS_CHANNEL_ID` | معرّف قناة الأعضاء `-100...` |
| `TRONGRID_API_KEY` | *(اختياري)* مفتاح من trongrid.io لرفع حد الطلبات |
| `ORDER_TTL_MIN` | *(اختياري)* صلاحية الطلب بالدقائق — الافتراضي 120 |

3. Start Command: `node server.js` (Railway يكتشفه تلقائيًا من package.json).
4. انشر — الموقع الحي يعمل فورًا على نفس رابطك.

## 3) دورة البيع الكاملة (آلية 100%)

1. العميل يختار المؤشر → الباقة → يظهر **مبلغ فريد** (مثل 179.37) + محفظتك + QR + زر «افتح البوت».
2. يضغط زر البوت → `/start ARK-XXXX` يربط حسابه بالطلب تلقائيًا.
3. يحوّل USDT → السيرفر يفحص TronGrid كل 20 ثانية ويطابق المبلغ الفريد.
4. عند التأكيد تلقائيًا:
   - البوت يرسل للعميل ✅ + **رابط دعوة لمرة واحدة** لقناة الأعضاء.
   - الموقع نفسه يعرض «تم تأكيد دفعتك» + رابط القناة.
   - **يصلك أنت** إشعار بالطلب + TX hash.
5. العميل يرسل اسمه في TradingView → البوت يستلمه ويرسله لك → تضيفه في Invite-Only Scripts (الخطوة اليدوية الوحيدة — TradingView لا يوفر API لها).

## 4) اختبار قبل الإطلاق

- افتح `https://رابطك/health` → يجب أن يرجع `{"ok":true,...}`.
- أنشئ طلبًا تجريبيًا من المتجر وافتح البوت — يجب أن تصلك رسالة الطلب.
- حوّل لنفسك مبلغًا صغيرًا (مثلًا أنشئ باقة تجريبية بـ 1 USDT مؤقتًا في `indicators` داخل index.html) وتأكد من اكتمال الدورة.

## 5) ملاحظات تشغيلية

- الطلبات تُحفظ في `orders.json` — عند **إعادة النشر** على Railway يُمسح الملف. الطلبات المدفوعة المؤكدة تكون وصلت للعميل والبوت قبل ذلك، لكن عند نمو المبيعات انقل التخزين إلى **Railway PostgreSQL** (أخبرني وأجهّز لك النسخة).
- المبالغ الفريدة لا تتكرر بين الطلبات المعلّقة — لا تعارض حتى مع طلبات متزامنة.
- الأسعار والباقات كلها في `public/index.html` داخل مصفوفة `indicators` — عدّل وانشر.
- لا تضع التوكن أو عنوان المحفظة داخل الكود — كلها من Variables.
